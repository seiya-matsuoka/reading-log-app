# 読書記録アプリ｜ドキュメント索引

> このディレクトリ（`/docs`）には、読書記録アプリの設計ドキュメントを Markdown でまとめています。  
> README では概要のみを扱い、詳細は本ディレクトリの各ドキュメントに記載している構成です。

---

## 読む順（おすすめ）

1. **仕様まとめ**：アプリで何ができるか（MVP仕様）
2. **企画・要件定義**：方針／制約（API・DB・SQLレイヤ・デプロイ含む）
3. **基本設計**：画面動線／データモデル／API一覧など全体設計
4. **詳細設計**：実装版の構成／各ファイル役割／バリデーション／ローディング・エラー体験
